# Mostro + MostriX

## Mostro

[Mostro](https://github.com/MostroP2P/mostro) is a P2P Bitcoin trading daemon over the Nostr protocol. It connects to lnd1 over gRPC and to Nostr relays over WebSocket.

### Docker image

Mostro is built from source inside Docker — no local Rust required. The Dockerfile lives at `docker/mostro/Dockerfile`:

1. Clones `https://github.com/MostroP2P/mostro` at `MOSTRO_REF` (default: `main`)
2. Compiles with `cargo build --release` inside `rust:1.86`
3. Copies the binary to a slim `debian:bookworm-slim` runtime image

The build takes ~10 minutes on first run. To build a specific version:

```bash
# .env
MOSTRO_REF=v0.17.4    # or any branch/tag
```

### Nostr key

Mostro needs a Nostr identity (`nsec` private key). Three options in priority order:

| Option | How |
|--------|-----|
| `.env` | Set `MOSTRO_NSEC_PRIVKEY=nsec1...` — fully unattended |
| Interactive | Paste your `nsec1...` key when the script prompts |
| Auto-generate | Press Enter — [rana](https://github.com/grunch/rana) is built in Docker and generates a new keypair |

The private key is saved to `mostro/nostr-private.txt` (chmod 600). The public key is shown at the end of setup and saved to `summary.txt`.

**Back up `mostro/nostr-private.txt`** — this is Mostro's identity on the Nostr network.

### Configuration

`mostro/settings.toml` is generated by `setup.sh`. Key fields:

```toml
[lightning]
lnd_grpc_host = 'https://127.0.0.1:10009'   # lnd1 gRPC
lnd_cert_file = '/config/lnd/tls.cert'
lnd_macaroon_file = '/config/lnd/admin.macaroon'

[nostr]
nsec_privkey = 'nsec1...'
relays = ['wss://nos.lol', 'wss://relay.mostro.network']

[mostro]
fee = 0
max_order_amount = 1000000    # sats
```

### Logs

```bash
mostro-logs
# or
cd ~/Mostro && docker compose logs -f mostro
```

## MostriX

[MostriX](https://github.com/MostroP2P/mostrix) is the terminal UI client for Mostro. It lets you browse orders, create trades, and manage admin disputes.

### Docker image

Built from source inside Docker — same pattern as Mostro. `docker/mostrix/Dockerfile`:

1. Clones `https://github.com/MostroP2P/mostrix` at `MOSTRIX_REF` (default: `main`)
2. Compiles with `cargo build --release` inside `rust:1.86`
3. Binary in final `debian:bookworm-slim` image

```bash
# .env — use a specific version
MOSTRIX_REF=v0.5.0
```

### Running MostriX

The `mostrix` alias installed by `setup.sh`:

```bash
mostrix
```

This runs: `docker run -it --rm --network host -v ~/.mostrix:/root/.mostrix mostrix:local`

The `~/.mostrix/` directory is mounted so the container reads the config and writes its database to the host.

### Key bindings

| Key | Action |
|-----|--------|
| Left / Right | Switch tabs |
| Up / Down | Navigate lists |
| Enter | Select / confirm |
| Q | Quit |
| M | Toggle User / Admin mode |
| C | Copy invoice to clipboard |
| Esc | Cancel / close popup |

### Configuration

`~/.mostrix/settings.toml` is generated by `setup.sh`:

```toml
mostro_pubkey = "<hex pubkey of Mostro>"
nsec_privkey  = "<your nsec key>"
admin_privkey = "<your nsec key>"
relays = ["wss://nos.lol", "wss://relay.mostro.network"]
user_mode = "admin"
```

The database is at `~/.mostrix/mostrix.db` (SQLite).
